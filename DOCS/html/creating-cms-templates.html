* Create a new html file
* Switch the the Editor view, write a filename in the "element" field.
* Click the selectbox and change the option to "html". Using html-content option disallows usage of any <head>, <body> and <html> tags.
* Activate tinyMCE.
* Create any layout.

* Click edit-area, choose CMS tags, add tags as desired.
* In editarea, you can see how Tinymce created style tags in the html.
* If we like, we can automatically convert these style tags to class tags with an external css file.
Be sure to save the HTML file.
Go to the Structure menu, hover over the saved HTML file, click "convert inline css to CSS stylesheet"

TinyMCE will automatically load the new css file in the editor, but there will not be any new-link rel stylesheet- added to the html file. Instead, this must be added manually.


Now if using certain CMSes to include certain CMS tags in templates, it may be possible that TINYMCE bails out and messes the tags (which could be caused by wrong freedomeditor tinyMCE settings aswell as lack of a good working tinyMCE codeprotector plugin.)
Therefore _always_ keep backup copies of HTML documents without the CMS template tags.


To manually load css files into tinyMCE, simply click the square with "css" in it, to the right of a file in the structure menu.
To reset the list of css files tinyMCE currently loads, click the CSS square in the bottom of the structure view, above the Merge option.
